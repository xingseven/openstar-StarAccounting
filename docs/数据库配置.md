# 数据库配置（PostgreSQL + Prisma）

## 目标

- 以 PostgreSQL 作为主数据库（符合开发框架要求）
- 后端通过 Prisma Migration 管理表结构

## 方式一：本机安装 PostgreSQL（推荐）

1. 安装 PostgreSQL（Windows 建议使用官方安装包）
2. 创建数据库（示例名：`openstar_Star Accounting`）
3. 在 `src/server/` 目录下创建 `.env`，配置：

```env
DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/openstar_Star Accounting?schema=public"
PORT=3004
PUBLIC_IP="0.0.0.0"
```

4. 生成 Prisma Client：

```bash
cd src/server
npm run prisma:generate
```

5. 执行迁移（首次会创建 migrations 并建表）：

```bash
cd src/server
npm run prisma:migrate
```

## 方式二：使用云数据库（Supabase/Neon）

1. 创建 PostgreSQL 实例并获取连接串（必须是 `postgresql://...`）
2. 将连接串写入 `src/server/.env` 的 `DATABASE_URL`
3. 执行：

```bash
cd src/server
npm run prisma:generate
npm run prisma:migrate
```

## 方式三：Docker Compose（最快）

1. 根目录下已有 `docker-compose.yml`
2. 启动数据库（MySQL）：
   ```bash
   docker-compose up -d
   ```
3. 在 `src/server/` 下创建 `.env`：
   ```env
   DATABASE_URL="mysql://root:Admin123.@localhost:3306/xf_dashboard"
   PORT=3004
   PUBLIC_IP="0.0.0.0"
   JWT_SECRET="dev-secret"
   ALLOW_DEV_HEADERS=1
   ```
4. 执行迁移：
   ```bash
   cd src/server
   npx prisma migrate dev --name init_mysql
   ```

5. 一键创建数据表（推荐）：
   ```bash
   cd src/server
   npm run db:create
   ```
   这会自动创建数据库（如果不存在）并创建所有表结构。

## 注意

- `.env` 不会提交到 GitHub，请使用 `.env.example` 作为模板
- 未配置 `DATABASE_URL` 时，后端会回退到“内存模式”，便于快速跑通流程，但数据不会持久化
