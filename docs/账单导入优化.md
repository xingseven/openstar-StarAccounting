# 账单导入优化 - 开发文档

## 目标

实现微信/支付宝账单的智能识别与统一分类。

## 问题分析

### 问题1：账单前面有多余行

微信/支付宝导出的CSV文件前面都有一些非数据行，需要动态识别列名所在行。

**示例 - 微信账单开头：**
```
微信账单...
交易时间  交易类型  交易对方  商品  收/支  金额(元)  支付方式  当前状态  交易单号  商户单号  备注
2026-01-01 00:00:00  商户消费  xxx  支出  100.00  零钱  支付成功  xxx  xxx
```

**示例 - 支付宝账单开头：**
```
alipay...
交易时间  交易分类  交易对方  对方账号  商品说明  收/支  金额  收/付款方式  交易状态  交易订单号  商家订单号
2026-01-01 00:00:00  餐饮美食  xxx  xxx  支出  100.00  余额宝  交易成功  xxx  xxx
```

**解决方案**：后端逐行扫描，识别列名行，从下一行开始解析数据。

### 问题2：两种账单类型不一致

| 维度 | 微信 | 支付宝 |
|------|------|--------|
| 分类字段 | 交易类型 | 交易分类 |
| 状态字段 | 当前状态 | 交易状态 |
| 收/支字段 | 收/支 | 收/支 |

## 技术实现

### 后端修改

1. **动态识别列名行** (`server/src/etl/importCsv.ts`)
   - 逐行扫描CSV，识别已知列名（交易时间、金额等）
   - 返回列名行索引，从下一行开始解析

2. **平台识别与字段映射** (`server/src/etl/mapTransaction.ts`)
   - 根据列名判断平台（微信/支付宝）
   - 动态映射字段位置

3. **分类映射** (`server/src/etl/categoryMap.ts`)
   - 创建微信交易类型 → 统一分类的映射
   - 创建状态映射

### 前端修改

1. **导入组件** (`web/src/app/(dashboard)/connections/page.tsx`)
   - 保留平台自动识别
   - 显示识别结果供用户确认

## 数据流

```
CSV文件上传
    ↓
后端接收文件流
    ↓
逐行扫描识别列名行（动态）
    ↓
判断平台（微信/支付宝）
    ↓
按平台映射字段
    ↓
应用分类/状态映射
    ↓
统一格式存储到数据库
    ↓
返回导入结果
```
