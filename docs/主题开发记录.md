# 主题开发记录

更新时间：2026-03-24（已补充收尾验收）

## 1. 目标

本轮主题改造的目标是把 dashboard 区域从“逐页手写样式”推进到“共享主题系统 + 全局切换 + 共享 UI primitive”。

最终要求：

- 新增主题时不再逐页重写页面
- 主题可以全局切换
- 布局、导航、卡片、表格、弹窗、表单尽量共用一套基础组件

## 2. 当前主题系统结构

核心文件：

- `web/src/themes/registry.ts`
  - 主题注册中心
  - 当前已内置 `default`、`graphite`、`spruce`、`terracotta`

## 7. 如何新增一个新主题

推荐流程：

1. 在 `web/src/themes/registry.ts` 新增主题定义
2. 补齐该主题的 `vars`
3. 主题中心会自动出现该主题
4. 已接入 `theme-primitives` 的页面会自动随主题切换

不要再做的事情：

- 不要直接逐页复制改颜色
- 不要再给每个页面重新写一套卡片/表单壳子
