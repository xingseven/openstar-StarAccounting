# 主题开发记录

更新时间：2026-03-24（已补充收尾验收）

## 1. 目标

本轮主题改造的目标是把 dashboard 区域从“逐页手写样式”推进到“共享主题系统 + 全局切换 + 共享 UI primitive”。

最终要求：

- 新增主题时不再逐页重写页面
- 主题可以全局切换
- 布局、导航、卡片、表格、弹窗、表单尽量共用一套基础组件

## 2. 当前主题系统结构

核心文件：

- `web/src/themes/registry.ts`

## 7. 如何新增一个新主题

推荐流程：

1. 在 `web/src/themes/registry.ts` 新增主题定义
2. 补齐该主题的 `vars`
3. 主题中心会自动出现该主题
4. 已接入 `theme-primitives` 的页面会自动随主题切换

不要再做的事情：

- 不要直接逐页复制改颜色
- 不要再给每个页面重新写一套卡片/表单壳子

## 11. 2026-03-28 模块色板分层增强

### 背景

当前默认主题已经具备统一的玻璃感与中性表面，但模块之间共用了过于接近的底色、面板层级和主操作色，导致借贷、储蓄、资产、消费在首屏上的识别差异不足。

### 本次调整

- 在 `theme-primitives` 中新增模块级色板变量：
  - `--module-surface-tint`
  - `--module-hero-tint`
  - `--module-metric-tint`
  - `--module-accent-strong`
  - `--module-progress-gradient`
- 通过 `getThemeModuleStyle()` 给核心模块挂接独立色板，而不是继续把差异化逻辑散落到每个页面里。
- 共享 `ThemeSurface / ThemeHero / ThemeMetricCard` 改为自动读取模块色板变量，从结构层直接拉开模块反差。

### 模块分色结果

- 仪表盘：冷蓝，保留总览与分析感。
- 消费：青绿，兼容微信/支付宝的双平台语义。
- 资产：靛紫，强调组合和持仓结构。
- 贷款：暖铜，强化压力与偿付语义。
- 储蓄：翡翠，突出增长和积累感。

### 验证

- `npm run typecheck` 通过。
- 定向 `npm run lint -- ...` 通过。
- `npm run build` 通过。

## 12. 2026-03-28 去毛玻璃化收敛

### 调整原因

模块分色落地后，页面识别度虽然提升，但共享主题底层仍然保留了较强的毛玻璃气质，包括：

- 半透明壳层与表面背景。
- Hero 区的模糊光斑和浮层。
- 局部 `backdrop-blur` 的筛选浮框与信息卡。

这些元素会继续把界面往“轻飘、悬浮”的方向推，和当前希望的更扎实、更朴素的财务工作台气质不一致。

### 本次收敛

- 主题注册表中的 light/dark 主题统一改为实色壳层、实色表面和更明确的边框。
- `ThemeHero / ThemeSurface / ThemeMetricCard` 去掉模块色渐变叠层，只保留纯色底和模块色边框。
- 仪表盘、消费、资产、贷款、储蓄页移除主要模糊装饰，压低视觉噪声。

### 当前结论

- 模块分色保留。
- 毛玻璃感大幅降低。
- 整体更接近“实色层级 + 模块强调色”的财务面板风格。
