# openstar Star Accounting - V2 高级功能开发文档

## 文档概述
本文档基于项目当前进度（核心业务已跑通），规划了系统向**智能化、高可用架构、精细化运营**演进的开发任务。每次开发完成一部分后，将同步更新 `docs/开发进度.md` 并记录 `CHANGELOG.md`。

---

## 阶段一：预算系统与预警增强 (Budget & Alerting)
**目标**：实现多维度预算控制与实时的预算超支预警机制。
- [ ] **数据库更新**：修改 `Budget` 模型，新增 `scopeType` (全局/分类/平台) 和 `alertPercent` (预警阈值，默认 80%)。
- [ ] **后端接口**：扩展 `/api/budgets`，支持根据阈值返回当前预算的健康状态（正常、警告、超支）。
- [ ] **前端展示**：
  - 预算进度条根据状态变色（<80%绿色，80%-100%黄色，>100%红色）。
  - Dashboard 首页新增“预算预警”提示卡片。

## 阶段二：自动化运维与端到端测试 (CI/CD & E2E)
**目标**：提升代码质量与部署效率，建立企业级 DevOps 流程。
- [ ] **CI/CD 流水线**：配置 `.github/workflows/ci.yml`，实现代码 Push 后的自动 Lint、单元测试运行。
- [ ] **E2E 测试集成**：引入 Playwright，编写核心链路测试用例（登录 -> 导入账单 -> 资产核算）。
- [ ] **Docker 部署优化**：完善 `docker-compose.yml`，加入健康检查 (Healthcheck) 与生产环境 Nginx 反向代理配置。

## 阶段三：数据批量操作与异常处理 (Data Management)
**目标**：提升大量账单数据的处理效率，降低错误数据的排查成本。
- [ ] **数据库更新**：新增 `ImportErrorLog` 模型，用于记录账单导入时的异常行数据与失败原因。
- [ ] **批量操作 API**：新增后端 `/api/transactions/batch` 接口，支持批量删除和批量修改分类。
- [ ] **前端交互**：
  - 消费明细列表支持多选操作（复选框）。
  - 新增“异常数据导入日志”查看与手动纠错面板。

## 阶段四：高级权限与后台管理 (Admin & RBAC)
**目标**：为系统提供管理员视角的管控能力。
- [x] **权限模型**：`User` 模型新增 `role` 字段 (USER / ADMIN)。
- [x] **鉴权中间件**：后端新增 `requireAdmin` 守卫，保护管理员接口。
- [x] **后台管理面板 (Admin Dashboard)**：
  - 新增 `/admin` 路由组。
  - 实现全站用户统计、系统运行状态查看、全局导入成功率看板。

## 阶段四补充：多账户数据隔离系统 (Multi-Account)
**目标**：支持家庭共享场景，多用户各自账单数据隔离。
- [x] **数据库层**：
  - 新增 `account` 表 - 账户信息
  - 新增 `account_member` 表 - 成员关系与权限（OWNER/ADMIN/MEMBER）
  - 所有业务表新增 `accountId` 字段
  - 用户 `defaultAccountId` 关联默认账户
- [x] **权限控制**：
  - `canViewOwn`/`canManageOwn` - 成员管理自己数据
  - `canViewAll`/`canManageAll` - 成员管理他人数据（需账户所有者授权）
- [x] **后端支持**：
  - `requireAccountId` 中间件自动获取用户默认账户
  - 导入数据时自动使用 accountId 隔离

## 阶段五：智能化与架构演进 (AI & Architecture) - 长期规划
**目标**：引入 AI 辅助功能与处理大规模数据的架构。
- [ ] **NLP 自动分类**：账单导入时，利用大模型或 NLP 提取商家名称与备注特征，自动映射到标准分类。
- [ ] **异常消费检测**：识别大额异常支出或高频低额支出，生成风险账单报告。
- [ ] **事件总线解耦**：引入 RabbitMQ/Redis 队列，将耗时的 ETL 导入流程异步化，提升前端响应速度。
