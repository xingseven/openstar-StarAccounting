# 系统优化与重构清单 (Optimization Plan)

本文档旨在记录 `v1.8.x` 后续的系统优化、代码重构与 UI/UX 一致性提升计划。

## 1. 代码复用与工程化 (Code Refactoring)

旨在减少重复代码，提升维护效率。

- [x] **抽离 `DelayedRender` 组件**
  - 目前 `ConsumptionDefaultTheme.tsx` 和 `DefaultSavings.tsx` 中均有重复定义。
  - **目标**: 提取至 `web/src/components/shared/DelayedRender.tsx`。
- [x] **统一 Skeleton (骨架屏) 组件**
  - **目标**: 创建 `web/src/components/shared/Skeletons.tsx`。
  - 包含: `ChartSkeleton` (图表占位), `ListTableSkeleton` (列表占位), `StatsCardSkeleton` (卡片占位), `CardListSkeleton` (简单列表占位)。
- [x] **创建通用 `EmptyState` 组件**
  - **目标**: 封装统一的空状态展示组件 `<EmptyState icon={...} title=”...” description=”...” />`。
  - 应用于: 资产、贷款、储蓄、消费等页面的无数据状态。
- [x] **类型定义集中管理**
  - **目标**: 将分散在各组件中的 `Asset`, `Loan`, `SavingsGoal` 等类型定义提取至 `web/src/types/` 或统一从 Prisma Client 导出。

## 2. 资产 (Assets) 与贷款 (Loans) 页面优化

旨在对齐”消费”与”储蓄”页面的高品质交互体验。

- [x] **添加骨架屏加载态**
  - 现状: 加载时直接白屏或显示 Loading 文本。
  - **目标**: 引入 `DelayedRender` 和通用 `Skeleton` 组件，实现平滑加载。
- [x] **统一页面布局参数**
  - **目标**: 统一容器间距 (`space-y-4 md:space-y-6`) 和最大宽度 (`max-w-[1600px]`)，保持全站视觉一致性。
- [x] **图标标准化**
  - **目标**: 将资产/贷款卡片中的支付宝/微信等图标替换为 `simple-icons` 官方品牌矢量图标（与消费页一致）。
- [x] **移除硬编码颜色**
  - **目标**: 审查并替换代码中的 Hex 颜色硬编码，改用 Tailwind 类名或 CSS 变量，适配主题系统。

## 3. 仪表盘 (Dashboard) 优化

- [x] **加载体验升级**
  - **目标**: 替换首页简单的 Loading Spinner 为与卡片布局一致的骨架屏，减少布局跳变。
- [x] **图表渲染优化**
  - **目标**: 为首页图表引入 `DelayedRender`，实现渐次入场动画。

## 4. 性能与交互 (Performance & UX)

- [x] **推广”吸顶筛选栏”模式**
  - **目标**: 评估在”贷款管理”或”资产管理”长列表页面应用 `FixedStickyHeader` 交互模式。
  - **评估结果**: 不适用 —— 资产和贷款页面目前没有复杂的筛选条件（搜索/平台/分类筛选），仅有简单的列表展示，无需吸顶筛选栏。消费页面因有多个筛选器才需要此功能。
- [x] **移动端适配检查**
  - **目标**: 专项测试 Assets 和 Loans 页面在移动端（<768px）的 Grid 布局表现，优化列数切换体验。
- [x] **滚动条抖动终极治理**
  - [x] (已完成) 全局应用 `scrollbar-gutter: stable`，防止页面内容因滚动条显隐而发生水平位移。

## 优先级建议

1.  **P0**: 抽离公共组件 (`DelayedRender`, `Skeletons`) —— 这是后续页面优化的基础。✅ 已完成
2.  **P1**: 优化 Assets 和 Loans 页面 —— 提升整体一致性。✅ 已完成
3.  **P2**: Dashboard 体验升级。✅ 已完成
