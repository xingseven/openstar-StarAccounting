# OpenStar 前端目录收口开发文档

## 0. 文档版本

- 大版本：`v1.0`
- 小版本：`v1.0.2`
- 本次更新：完成旧前端目录删除后，再将后端从 `src/server/` 收口到根目录 `server/`，确认仓库仅保留 `web/` 前端与 `server/` 后端。

---

## 1. 背景

仓库此前同时存在两套前端：

- 生效前端：`web/src`
- 历史前端副本：`src` 下除 `server` 外的前端目录

同时根目录还保留了一套旧前端配置：

- `components.json`
- `next.config.ts`
- `tsconfig.json`
- `next-env.d.ts`
- `public/`

如果继续保留这两套路径，会带来三类问题：

1. 新代码容易误写到旧目录。
2. 构建、脚手架和部署容易误指向旧前端。
3. 旧前端中的临时修正可能无法及时合并回当前主线。

## 3. 实际迁移结果

### 3.1 已确认无需迁移的差异

以下类别已确认以 `web/src` 为主，不再保留 `src` 版本：

- 总览路由与主题接线层
- 主题注册表与主题预览清单
- 共享导航、侧边栏、移动端底栏和 Header
- Dashboard、Consumption、Assets、Loans、Savings 等主题页面实现
- 旧的 `src/app/(dashboard)/template.tsx` 简单包裹层

### 3.2 从 `src` 迁入 `web` 的有效差异

本轮实际迁入 `web` 的差异如下：

1. `web/src/components/shared/AuthGate.tsx`
   - 补回 `setAuthUser`，允许登录/注册成功后直接注入用户缓存。
2. `web/src/app/auth/login/page.tsx`
   - 登录成功后调用 `setAuthUser(data.user)`。
   - 新增 `router.prefetch(next)`，在登录过程中预取跳转目标页。
3. `web/src/app/auth/register/page.tsx`
   - 注册成功后调用 `setAuthUser(data.user)`。

这部分是本轮确认仍对当前主线有价值、且可以安全合并到 `web` 的旧前端差异。

## 5. 当前仓库状态

当前仓库结构已经收口为：

1. `web/`：唯一前端工程
2. `server/`：唯一后端工程
3. 根目录：仅保留启动脚本、部署脚本、文档和辅助文件

这意味着：

1. 新成员不会再把前端代码误写到旧目录。
2. 根目录不再存在会误导人的旧前端入口。
3. 前端验证、构建、部署和功能开发都以 `web/` 为唯一目标。

---

## 6. 验收标准

本次收口完成后，已满足以下条件：

1. 仓库内只保留 `web/` 这一套前端源码。
2. `src/` 仅保留 `server/` 后端。
3. 旧前端中的有效差异已合并到 `web/`。
4. 根目录旧前端配置已移除，不再存在误用入口。
