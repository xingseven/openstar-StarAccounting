# 更新发布与镜像说明

## 1. 目标

本项目的更新方案采用：

1. 网站镜像优先
2. GitHub 备用
3. 关于页面只请求本项目后端
4. 用户下载更新时不跳转到 GitHub 或外部网站

也就是说：

- 用户打开关于页时，前端只访问你自己的 `/api/update/check`
- 用户点击下载安装时，前端只访问你自己的 `/api/update/download/...`
- 后端再决定去本地文件、你的网站镜像，或者 GitHub 获取资源

## 2. 本地更新清单

项目默认会优先读取：

`web/public/updates/latest.json`

你已经可以把它理解成“版本发布清单”。

这个文件里定义：

- 最新版本号
- 发布时间
- 更新说明
- 网页版更新方式
- App 安装包下载列表

## 3. 网站镜像怎么放

如果你想让没有代理的用户也能更新，最简单的方式是：

1. 把安装包放到：

`web/public/updates/downloads/`

2. 在 `web/public/updates/latest.json` 中填写对应下载信息

这样部署后，文件会跟网站一起被访问到，不需要额外跳 GitHub。

例如你后面可以放：

- `web/public/updates/downloads/openstar-android-2.3.8.apk`
- `web/public/updates/downloads/openstar-web-2.3.8.zip`

## 4. GitHub 备用镜像

如果你还想保留 GitHub 作为备用更新源，可以做两件事：

### 4.1 GitHub 上也放一份 `latest.json`

推荐路径：

`web/public/updates/latest.json`

然后通过 Raw 地址访问，例如：

```text
https://raw.githubusercontent.com/<owner>/<repo>/main/web/public/updates/latest.json
```

### 4.2 安装包放到 GitHub Release

例如：

- Android APK
- Windows EXE / ZIP
- Web 离线包 ZIP

然后把下载 URL 填进 `latest.json`

## 5. 后端如何工作

当前后端已经支持：

- `GET /api/update/check`
- `GET /api/update/download/:channel/:downloadId`

逻辑是：

1. 优先读取本地 `web/public/updates/latest.json`
2. 如果本地没有，再尝试远程更新源
3. 下载时通过后端代理/转发，不让前端直接跳外链

## 6. 环境变量

可选环境变量：

```env
UPDATE_LOCAL_MANIFEST_PATH="web/public/updates/latest.json"
UPDATE_MANIFEST_URLS="https://your-domain.com/updates/latest.json,https://raw.githubusercontent.com/owner/repo/main/web/public/updates/latest.json"
```

说明：

- `UPDATE_LOCAL_MANIFEST_PATH`
  用于指定本地更新清单路径
- `UPDATE_MANIFEST_URLS`
  用于配置远程备用更新源，多个地址用逗号分隔

## 7. 关于页如何更新

关于页里：

- 网页版更新
  - 通过“刷新并更新”生效
  - 不需要下载安装包
- App 更新
  - 通过下载安装包更新
  - 当前设计默认是重新安装

所以你当前的理解是对的：

- 网页版：刷新获取新资源
- App：下载新包重新安装

## 8. 以后你发新版时怎么做

每次发版本，建议按下面顺序：

1. 先更新 `CHANGELOG.md`
2. 更新 `web/public/updates/latest.json`
3. 把安装包放进 `web/public/updates/downloads/` 或 GitHub Release
4. 部署网站
5. 在关于页点击“检查更新”验证

## 9. 你现在最需要做的事

如果你准备开始真的发“移动端更新包”，建议你下一步只做这两件事：

1. 先决定 App 的安装包格式
   - Android：`.apk`
   - Windows：`.exe` 或 `.zip`
2. 把第一份安装包放进
   - `web/public/updates/downloads/`

只要安装包文件真的放进去了，关于页的下载更新链路就能直接用起来。
